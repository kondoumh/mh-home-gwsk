Update 2009/02/11

また2年越しのバージョンアップです。アーリーアクセス版の最終版からいくつか不具合修正をしていますので、最新版1.80をご利用下さい。

今回は、ファイル形式を新しくしたことで、かなりの機能を追加することができました。古い形式も読み書きできるようにしています。OSに2種類のiEditファイル形式が登録されてしまいますが、ご了承下さい。

Vistaをようやく使い始めたので、ファイルタイプのレジストリーへの登録の部分も見直しました。管理者モードでの起動をしなくても、ファイルタイプを登録できるようになっているはずです。また、アイコンも24bitカラーにしてリニューアルしました。どこかで見たようなデザインになってしまいました。

Update 2007/11/27

バージョンアップ1年以上ぶりです。アーリーアクセス版で随時機能を追加していたのですが、だいぶ更新がたまってきて、ずるずる行くのもなんなのでこのあたりでリリースということにしました。

このバージョンから開発環境をVisual C++ 8.0(VS2005)からVisual C++ 9.0(VS2008)にスイッチしています。あと、Win98での動作確認、もうやめようと思います。

うぃんどうずきゅーじゅーはち? 。。旧世紀のOSですな。。

主な機能追加は以下の通りです

  • 書式設定ツールバーで簡単にノードの属性を編集できるようになった。
  • アウトラインビューで、ネットワークビューでの描画順を制御できるようになった。
  • ノードの簡易グループ化機能
  • ノード・リンクを複数選択して削除することが可能になった。
  • ノードの自動リサイズを改善した

描画順の制御とノードの簡易グループ化機能を組み合わせると、面白い使い方ができます。詳しくは、はてなダイアリーにて。

Update 2006/09/22

今回は、ノード追加・リンク追加やよく行うノードの編集作業を楽にすることを中心に改修しました。ツールバーも少し変わっています。

マウスホイールボタンによるリンク作成機能を拡張して、ノード追加とリンク作成を同時に行えるようにしたのですが、これはUMＬモデリングツールEnterprise Architect にインスパイアされた をパクった ものです。詳しくは改訂履歴と、マニュアルをご覧下さい。

あと、フォントサイズを大きく/小さくするボタンやリンクの矢印や線をあらかじめ指定してから作成する機能などです。ユーザーズマニュアルでの該当箇所は以下の通りです。

  • マウス・ホイールボタンによるリンク／関連ノードの追加

  • ラベルのみのノード作成ボタン

  • マルチ行、シングル行設定ボタン

このバージョンから開発環境をVisual C++ 6.0(SP6)からVisual C++ 2005にスイッチしています。ツールバーのボタンが増えて開発環境も変わったので、マイナーバージョンをインクリメントしました。

Update 2005/03/22

ごぶさたしております。今回は、少し機能を追加してリリースすることができました。

マウスの中ボタンでリンクを張ることができるようにしたのは、某分析系ツールの操作性に影響を受けたものです。

芋づるモードは、自動レイアウト機能の応用系です。芋づるモードに関しては、KJ法っぽいことをやるときに使えるのではないかと思っています。既存のレイアウト機能だと図全体に影響が出てしまうので、自分でもあまり使っていませんでした。フローチャートのようなきっちりした図には適さないと思います。マウスの中ボタンでリンクが張れる機能と芋づる機能とで、かなり直感的にブレスト作業が行えるようになったのではないかと思います。

スタイルシートに関しては、貝塚さんにiEdit用のXSLを開発していただいたので、対応しました。なかなか見栄えのよいページになります。

マインドマップツールのようにINSキーでサブノードとリンクの追加、リターンキーで兄弟ノードの追加みたいなものを作るのもよいかもと思いました。今回実装した芋づるを応用すれば、ノードを追加したとたん自動でレイアウトもしてしまうというようなこともできそうな気がします。

今回の機能追加は、アウトラインビューとネットワークビューの連動方法にバリエーションを持たせたというのが、メインです。 詳しくはこちら。従来「全ての階層のノードを表示」するオプションがありましたが、文字通り全て表示されてしまい、使い道は今ひとつでした。今回は、サブ階層(サブツリー)のノードだけを表示するモードを提供したので、ちょっと使い方の幅が広がるかななんて思います。

Update 2002/09/30

クリップボードコピー処理では、描画領域の論理サイズに、係数をかけて必要なピクセル数を計算しています。Windowsのバージョンによってこの係数の調整が必要なのですが、同じWindows 2000でも時々サイズがおかしいケースがあります。そこで今回はメタファイルサイズを調整する機能(係数を変える機能)を追加しました。今までクリップボードコピーがうまくいかず使っていない方も調整してみてください。クリップボード系のプログラミング方法をもう少しちゃんと調べれば解消できる問題なのかもしれません。

Kazuyoshi Software CLUBさんの指摘により、ネットワークビューにおけるスクロール関係の不具合を修正しました。

SVG出力機能をつけてみました。SVG(Scalable Vector Graphics)は、Webブラウザ上でVector画像を扱える技術です。SVG自体はXMLデータで、プラグインが必要となりますが、WindowsのGDIと同程度の表現力はあるようです。SVGのプラグインは、Adobe社のダウンロードサイト から入手してください。iEditの図をiEdiｔを使っていない人に渡したいときなどに活用できるのではないかと思います。図の再現度は今一歩といったところですが

現在のところ、ラベルのアライメントとか複数行は無視されます。メタファイル(シェイプ)に関しては、単なる矩形になります。また、リンクの曲がり具合が弱かったりします。

今回のSVGのXMLデータを出力する部分はMSXMLを使用しました。今までもXMLのParseにはMSXMLを利用してますが、書き込みに使ったのは今回が初でした。MSXMLでDOMツリーを作って出力するのですが、DOCTYPEタグの埋め込み方が分からない、生成されたXMLファイルに改行文字がが入ってないなどの問題がありました。Xerces C++ などに乗り換えようかとも思います。

iEdit for Windows CE ?

最近、CompaqのiPAQ や 東芝のGENIO e のような魅力的な製品が出てきて、CE機に興味が出てきました。実は去年おシゴトでWindows CEのプログラムを組んだのですが、あまりいい印象がありませんでした。Windows CE ToolKit for Visual C++という製品を購入しVisual C++ に プラグインして使ったところ、Visual C++の環境がおかしくなるという問題が出ました。当時はPalmが元気だったのもあり、その後 CE は敬遠していました。ところが、最近 Windows CEの開発環境である eMVT(eMbedded Visual Tools)が無償で提供されていると知り、思わずダウンロードしてしまいました(300MB)。

eMVT は VisualC++ へのプラグインでなく独自のIDE(eMbedded Visual C++ 3.0)を提供しています。しかも操作体系はVisual C++ 6.0 とまったく同じです。Visual C++への影響もないようです。

そこで、CEベースで iEdit 的なアプリが書けないか少し試してみました。ノードの描画やマウス(スタイラスペン?)による移動は問題なく行えました。(実機を持っていないので付属のエミュレータで遊んでみただけですが)。高機能な開発ツールが無償で提供されたことで、今後Pocket PCやHandheld PCのアプリが増えてきそうです。

iEdit for Windows CEを作るとしたら、どのプラットフォームにしようか迷うところです。通勤電車で使うという用途でPocketPCが欲しいのですが、最近打合せが多いので、文字入力が楽なHandheld PCが便利かなあなんて思っています。

ちなみにiEdit for Java ？ のようなことも書いてましたが、今のところ着手予定は無いです。Javaのネイティブコードコンパイラが標準で提供されればMFCからJFCに乗り換えてもよいかなと思ってますが．．．。シゴトでJFC使いましたが動作がトロトロしてるので、日常使うツールには厳しいかなと思ってます。プログラムの最適化が足りないのでしょうか？ 開発はすんごく楽なんですがねえ。

iEditの利用方って？

ところで、iEditのユーザの方は、どのようにiEditを活用されているのでしょうか？

私自身は、主にドキュメント(設計書や作業報告書)の下書きツール兼テキストや図のデータベースとして使っています。おシゴト単位のツリーを作り、その中でテキストや図を管理します。本文の下書きを章立てのツリーで管理し、各章の下位階層に図を置きます。関連URLや打合せの議事録、重要なメールなどもそのツリーにくっつけます。

報告書を書く時は、サブツリーをテキストに出力し、テキスト部分だけをWordに貼り付け、次に図をクリップボード経由で順次貼り付けていきます。Wordの機能としては、改ページ挿入、ヘッダーフッターの設定、印刷の３つだけ覚えていればなんとかなります。図は「形式を選択して貼り付け」でペーストし拡張メタファイルとして貼り付けるのがミソです

別のシゴトで似たような図を作るときはすでにある図をコピペして使いまわします。そのときに、ツリーを縦断する検索機能で必要なノードやリンクを探しています。シゴト関係のファイルが基本的に１個で済むので楽です。

後輩におシゴトを引き継ぐときは、そのシゴトのサブツリーだけのファイルを渡してしまいます。したがって、必然的に後輩はiEditユーザになってしまってます。

iEditのようなソフトウェアを作成している私がこんなことを言うのもなんですが、文書の構成を考えるということに特化して考えると、アウトラインプロセッサって意外と使いにくいなと思います。使い慣れたテキストエディタの方が作業が速いのです。(Wordのアウトライン機能など言語道断。メモ帳の方がマシです)。

みなさまの中で面白い活用事例がありましたら、メールでもいただけるとうれしいです。

Update 2000/10/7

iEditをリリースしてからもうすぐ１年になります。ちょっとした思い付きで作った割には、けっこう使っていただいているようで驚いております。色々なご意見をお寄せくださった皆様に感謝いたします。

現在寄せられている要望で応えられそうなものをリストアップしてみます

  • 各ビューの背景色、文字色の設定
  • ノードに日付時刻属性を持たせて、ソート可能にする
  • ネットワークビューでノードの並べ替えコマンドの充実（タイル、カスケードなど）
  • リンクビューにリンク元、リンク先を表示する

ファイル形式の変更を迫られる内容も含まれています。時期の確約はできませんが、順次対応していきたいと思います。

この他に、「ネットワークビューの表示サイズを一般のドローツールと同様任意倍率にして欲しい」という要望は頻繁に寄せられます。これに関しては、ネットワークビューの描画アルゴリズムの大幅見直しが必要となるため、今のところ考えていません。

「プリンタに出るサイズが大きすぎる（小さすぎる）」という声も多いです。プリンタできれいに出力するには、ワープロにクリップボード経由で貼り付けて印刷してくださいと回答しています。

  • iEditでメールの読み書きができるようにする
  • Javaへの移植
    JFCのお勉強として、iEdit for Javaというのを作ってみたいです。iEditの基本機能＋iEditのXMLデータの読み書きを実現すれば、かなりの「お勉強」になると思ってます。iEditをLinuxやMacやSoralisでも使えるようになれば面白いですね。
